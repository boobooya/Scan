Per-Channel Knowledge File

      
knowledge/{channel_label}.json — One file per channel, plug directly into any AI model as context


      

        channel
        string
        YouTube channel @handle
      

      

        updated_at
        ISO8601
        Last extraction timestamp
      

      

        video_count
        number
        Videos processed so far
      

      

        claims[ ]
        array
        All extracted claims
      

      

        ↳ id
        string
        MD5 of claim text (dedup key)
      

      

        ↳ text
        string
        The extracted claim / insight
      

      

        ↳ p
        float 0–1
        Probability / confidence score from AI
      

      

        ↳ source_vid
        string
        YouTube video ID origin
      

      

        ↳ timestamp
        ISO8601
        When claim was extracted
      

    


    

      
Global Truth File

      
truth/truth.json — Cross-channel merged & ranked insights


      

        text
        string
        Deduplicated claim text
      

      

        p
        float 0–1
        Weighted average probability
      

      

        conf
        float 0–1
        Confidence (grows with mentions)
      

      

        n
        number
        Times this claim was seen across channels
      

      

        sources[ ]
        array
        Which channels contributed this claim

Using with AI Models

      
Drop the JSON directly as system context or a document block

      


# OpenAI / Claude / Gemini
import json

knowledge = json.load(open("knowledge/hubermanlab.json"))
claims_text = "\n".join([c["text"] for c in knowledge["claims"]])

system_prompt = f"""
You have access to the following expert knowledge
extracted from {knowledge['channel']}:

{claims_text}

Answer questions using this knowledge as grounding.
"""

      

    


  


  
  


    

      
Host on GitHub Pages

      
This page is a standalone HTML file — zero build step, zero dependencies.

      

        

          
1

          

            
Create a GitHub repo

            
Name it openclaw or anything you like. Make it public.

          

        

        

          
2

          

            
Push index.html to the root

            
That's the only file needed for the web app. The Python scripts are downloaded on demand from the Scripts tab.

          

        

        

          
3

          

            
Enable GitHub Pages

            
Settings → Pages → Source: main branch, / (root). Your app will be live at https://username.github.io/openclaw

          

        

        

          
4

          

            
Run Python backend locally

            
Download the scripts, drop config.json and channels.json next to them, then ./launch.sh. Knowledge files appear in knowledge/.

          

        

        

          
5

          

            
(Optional) Commit knowledge files to GitHub

            
Push the knowledge/ folder to your repo so others can fetch the JSON files directly via raw URLs — perfect for sharing AI context.

          

        

      

    


    

      
Automate with GitHub Actions

      
Run extraction on a schedule and auto-commit updated knowledge files

      

        

          .github/workflows/extract.yml
          
        


name: Extract Channel Knowledge
on:
  schedule:
    - cron: '0 6 * * *'  # Daily at 6am UTC
  workflow_dispatch:  # Manual trigger

jobs:
  extract:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install deps
        run: |
          pip install yt-dlp redis requests numpy
          sudo apt-get install -y redis-server ffmpeg
          sudo service redis-server start

      - name: Write config
        run: echo '${{ secrets.OPENCLAW_CONFIG }}' > config.json

      - name: Run extraction
        run: |
          python3 ingest.py
          python3 worker.py
          python3 truth.py

      - name: Commit knowledge files
        run: |
          git config user.name "openclaw-bot"
          git config user.email "bot@openclaw"
          git add knowledge/ truth/
          git diff --staged --quiet || git commit -m "🧠 Update knowledge $(date -u +%Y-%m-%d)"
          git push

      

      

        Store your config.json content (with API keys) as a GitHub Secret named OPENCLAW_CONFIG.
